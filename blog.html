Discuss the scope of var, let and const:

Scope determines the accessibility of variables. There are three types of scope in JavaScript.
They are

  • Block Scope
  • Function Scope
  • Global Scope

Block Scope:

If you declare a variable with let and const inside a {} block you cannot access it from outside the block. But if declare with var you can access it from outside the block. Because var is not a block scope where let and const are clock scope.

Global Scope:

Which variables declare outside of a function or block scope that is Global. Global variable can be accessed form anywhere in a JavaScript Program. If you declare a variable with var, let or const outside of a block scope or function scope that is global scope and you can access this variable from anywhere of JavaScript Program. So var, let and const they all are Global Scope.

Function Scope:

If you declare a variable with var, let or const in a function you cannot access from the outside of the function. Because every function create a new scope. So var, let and const they all are function scope.

The use cases of null and undefined:

Null:

Null in JavaScript means an empty value. Null is basically an assignment value given to a variable. when we assign null as a value to any variable, it means that it is empty or blank. It is show that the variable has no value.

Undefined:

Undefined means 'not defined'. If we declare a variable but do not assign a value to it, that called undefined.
